# NPM Package Management Guide

**Purpose:** Keep npm packages tidy, contained, and fully removable with zero trace.

---

## Philosophy: Clean Global Installs

**Goal:** Minimize global npm packages. Each installed package should be:
1. **Tracked** - Listed in `npm-global.txt`
2. **Justified** - Has a clear use case (CLIs, dev tools)
3. **Removable** - Can be uninstalled cleanly with `npm uninstall -g`

**Why minimize globals?**
- Global packages pollute `~/.npm` and `/usr/lib/node_modules`
- Hard to track what's installed vs. dependencies
- Easy to accumulate cruft over time
- Project-specific packages belong in `package.json`, not global

---

## Package Categories

### ✅ Good Candidates for Global Install
- **CLI tools** you use across projects: `typescript`, `tsx`, `npm-check-updates`
- **Development utilities**: `nodemon`, `serve`, `http-server`
- **Package managers**: Already using `pnpm`, `bun` (system-level via pacman/AUR)

### ❌ Bad Candidates for Global Install
- **Project dependencies**: `react`, `express`, `lodash` → Use `package.json`
- **Build tools**: `webpack`, `vite`, `rollup` → Use `package.json` + `npx`
- **Frameworks**: `next`, `nuxt`, `sveltekit` → Use `npx create-*` or project install
- **Linters/formatters**: `eslint`, `prettier` → Use `package.json` for per-project configs

## Installation Methods

### Method 1: Global Install (Persistent CLI Tools)
```bash
# Install globally
npm install -g <package>

# Update tracking list
npm list -g --depth=0 --json | jq -r '.dependencies | keys[]' | sort > ~/agents/sys-admin/package-management/npm/npm-global.txt

# Commit to git
cd ~/agents/sys-admin
git add package-management/npm/npm-global.txt
git commit -m "Install npm package: <package>"
git push
```

**When to use:**
- CLI tools you use daily (`typescript`, `tsx`)
- System-wide dev utilities (`nodemon`)

### Method 2: npx (One-off Execution, Zero Install)
```bash
# Run without installing
npx <package> [args]

# Examples:
npx create-react-app my-app
npx tsc --init
npx serve ./build
```

**When to use:**
- One-off commands (`create-*` scaffolding tools)
- Trying out tools before committing
- Tools you rarely use

### Method 3: Project-local Install (Per-project Dependencies)
```bash
# Install in project
cd /path/to/project
npm install <package>
# or: pnpm add <package>

# Package goes in package.json
```

**When to use:**
- Everything else (95% of packages)
- Project-specific dependencies
- Build tools, frameworks, libraries

## Package Tracking System

### Global Package List
**File:** `package-management/npm/npm-global.txt`
**Format:** One package per line, unsorted (order doesn't matter)
**Generated by:**
```bash
ls -1 /usr/lib/node_modules/ | while read pkg; do pacman -Qo "/usr/lib/node_modules/$pkg" >/dev/null 2>&1 || echo "$pkg"; done > ~/agents/sys-admin/package-management/npm/npm-global.txt
```

**How it works:** Lists all packages in `/usr/lib/node_modules/` that are NOT owned by any pacman package. This automatically excludes system packages (npm, pnpm, node-gyp, etc.) which are installed via pacman/AUR. Only user-installed npm global packages appear in the list.

**Example:**
```
http-server
nodemon
npm-check-updates
tsx
typescript
```

### Installed Packages Tracking
**File:** `package-management/npm/NPM_INSTALL_TRACKING.md`
**Format:** Checklist-style, similar to `PACKAGE_INSTALL_TRACKING.md`
**Purpose:** Document WHY each package is installed and what it's used for

## Clean Removal Process

### Step 1: Uninstall Package
```bash
npm uninstall -g <package>
```

### Step 2: Update Tracking List
```bash
ls -1 /usr/lib/node_modules/ | while read pkg; do pacman -Qo "/usr/lib/node_modules/$pkg" >/dev/null 2>&1 || echo "$pkg"; done > ~/agents/sys-admin/package-management/npm/npm-global.txt
```

### Step 3: Verify Clean Removal
```bash
# Check package is gone from node_modules
ls /usr/lib/node_modules/ | grep <package>  # Should return nothing

# Check no leftover files in cache
find ~/.npm -name "*<package>*"
```

### Step 4: Commit Changes
```bash
cd ~/agents/sys-admin
git add package-management/npm/npm-global.txt
git commit -m "Remove npm package: <package>"
git push
```

### Step 5: Update Tracking File
Edit `NPM_INSTALL_TRACKING.md` to move package from "Installed" to "Removed" section.

## Avoiding npm Bloat

### Use pnpm/bun Instead of npm
- **pnpm** - 3x faster, uses hard links (saves 50-90% disk space)
- **bun** - 10-100x faster, drop-in replacement for npm/node
- Both already installed via pacman/AUR

### Periodic Cleanup
```bash
# Clear npm cache (safe operation)
npm cache clean --force

# List global packages (user-installed only)
ls -1 /usr/lib/node_modules/ | while read pkg; do pacman -Qo "/usr/lib/node_modules/$pkg" >/dev/null 2>&1 || echo "$pkg"; done

# Remove unused globals
npm uninstall -g <package>
```

### Audit Global Packages Quarterly
```bash
# List all user-installed global packages
ls -1 /usr/lib/node_modules/ | while read pkg; do pacman -Qo "/usr/lib/node_modules/$pkg" >/dev/null 2>&1 || echo "$pkg"; done

# Ask yourself:
# - Have I used this in the last 3 months?
# - Could I use npx instead?
# - Does this belong in a project's package.json?
```

## Integration with /pacman Command

The `/pacman` slash command now:
1. Regenerates npm global package list (`npm-global.txt`)
2. Cross-references with `NPM_INSTALL_TRACKING.md`
3. Reports discrepancies (installed but not tracked, tracked but not installed)
4. Recommends packages to install/remove

## Best Practices

### 1. Think Twice Before Global Install
Ask yourself:
- **Do I need this globally?** → Or can I use `npx`?
- **Will I use this across multiple projects?** → Or just one?
- **Is this a CLI tool?** → Or a library dependency?

### 2. Prefer npx for One-offs
```bash
# ❌ Don't do this:
npm install -g create-react-app
create-react-app my-app
npm uninstall -g create-react-app

# ✅ Do this instead:
npx create-react-app my-app
```

### 3. Document Every Global Install
Update `NPM_INSTALL_TRACKING.md` immediately after installing globally.

### 4. Use pnpm for Projects
```bash
# In project directories, use pnpm instead of npm
pnpm install
pnpm add <package>
pnpm remove <package>
```

### 5. Keep Global List Minimal
**Target:** < 10 global packages
**Reality check:** If you have > 20, audit and remove

## npm vs pnpm vs bun

| Feature | npm | pnpm | bun |
|---------|-----|------|-----|
| Speed | Baseline | 3x faster | 10-100x faster |
| Disk Usage | Baseline | 50-90% savings | Minimal |
| Global Install | Yes | Yes | Yes |
| Project Install | Yes | **Preferred** | Experimental |
| Cache Strategy | Copies | Hard links | Native |
| Compatibility | 100% | 99% | 95% |

**Recommendation:**
- **npm** - Global CLI tools only (minimal use)
- **pnpm** - All project dependencies (primary package manager)
- **bun** - Fast script execution, testing new features

## Troubleshooting

### "npm WARN deprecated"
This is normal for old packages. Update with:
```bash
npm update -g <package>
```

### "Permission denied" on Global Install
**Don't use sudo!** npm is installed via pacman, should work without sudo.
If you see this, check npm prefix:
```bash
npm config get prefix  # Should be /usr
```

### "Cannot find module" After Uninstall
Package was removed cleanly. If error persists, clear cache:
```bash
npm cache clean --force
```

### Global Package List Out of Sync
Regenerate from actual system state:
```bash
ls -1 /usr/lib/node_modules/ | while read pkg; do pacman -Qo "/usr/lib/node_modules/$pkg" >/dev/null 2>&1 || echo "$pkg"; done > ~/agents/sys-admin/package-management/npm/npm-global.txt
```

## Notes

- **Scalable detection:** Uses `pacman -Qo` to identify which packages are system-installed vs npm-installed. No hardcoded package name filters.
- **Auto-update:** The `/pacman` command regenerates `npm-global.txt` automatically.
- **No sudo needed:** npm installed via pacman works without sudo for global installs.
- **pnpm preferred:** Use pnpm for all project dependencies, npm only for globals.
