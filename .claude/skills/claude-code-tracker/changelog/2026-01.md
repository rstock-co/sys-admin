## ✨ v2.1.27 - January 30, 2026

### What Changed
Debug logging for tool call failures and denials. New `--from-pr` flag to resume sessions linked to specific GitHub PRs. Sessions automatically link to PRs when created via `gh pr create`. Permissions now respect content-level `ask` over tool-level `allow`. Fixed `/context` command colored output display. Fixed status bar duplicating background task indicator when PR status displayed. VSCode: Enabled Claude in Chrome integration. VSCode: Fixed OAuth token expiration causing 401 errors after extended sessions. Windows: Fixed bash command execution failures for users with `.bashrc` files. Windows: Fixed console windows flashing when spawning child processes.

### What It Means
**--from-pr Flag:**
Resume a Claude Code session directly from a GitHub PR number or URL:
```bash
claude --from-pr 123
claude --from-pr https://github.com/org/repo/pull/123
```

**Automatic PR Linking:**
When you create a PR via `gh pr create` during a session, that session is now automatically linked to the PR.

**Debug Logging for Tool Calls:**
Tool failures and permission denials now appear in debug logs. Helpful for understanding why operations failed or were blocked.

## 🚀 v2.1.20 - January 27, 2026

### What Changed
PR review status indicator in prompt footer. Support for loading `CLAUDE.md` files from directories via `--add-dir` flag. Task deletion capability via TaskUpdate tool. Vim normal mode arrow key history navigation. External editor shortcut (Ctrl+G). Session compaction fixes. Wide character and Unicode improvements.

### What It Means
**--add-dir CLAUDE.md Loading:**
Load CLAUDE.md instructions from arbitrary directories. **Requires environment variable:**
```bash
export CLAUDE_CODE_ADDITIONAL_DIRECTORIES_CLAUDE_MD=1
```
Once enabled: reference shared instructions from a central location, layer multiple instruction sets.

**Note:** Currently undocumented. See [GitHub Issue #21138](https://github.com/anthropics/claude-code/issues/21138).

**Task Deletion:** TaskUpdate tool can now delete tasks, not just update them.

## 🚀 v2.1.16 - January 22, 2026

### What Changed
Native task management system with dependency tracking. VSCode plugin management. OAuth remote session browsing. Fixed out-of-memory crashes when resuming sessions with heavy subagent usage.

### What It Means
**Native Task System:**
First-class task management built into the core:
- **Dependency Tracking**: Tasks declare `blockedBy` and `blocks` relationships
- **Status Tracking**: `pending` → `in_progress` → `completed`
- **Agent Coordination**: Sub-agents can create, claim, and complete tasks

Session-scoped (don't persist across sessions by default). Need "hydration pattern" for cross-session continuity.

### How to Use It
```bash
# Use /tasks to view current task state
# Tasks are managed via TaskCreate, TaskUpdate, TaskGet, TaskList tools
```

## 🚀 v2.1.0 - January 9, 2026

### What Changed
Automatic skill hot-reload. `context: fork` for skill sub-agents. `language` setting. Shift+Enter out-of-box in iTerm2, WezTerm, Ghostty, Kitty. Vim motions (`;`, `,`, `y`, `p`, `>>`, `<<`, `J`). `/plan` command. Wildcard Bash tool permissions. Unified Ctrl+B backgrounding.

### What It Means
**Skill Hot-Reload:** Edit a skill, immediately available. No restart needed.

**context: fork:** Skills spawn sub-agents in forked context. Research without polluting main context. Experimental operations that might fail.

**Vim Motions:** Comprehensive Vim navigation in input. `;`/`,` repeat finds, `y` yank, `p` paste, `>>`/`<<` indent.

**Unified Ctrl+B:** Both bash commands AND agents can be backgrounded.

### How to Use It
```yaml
---
name: research-topic
context: fork
---
Research this topic thoroughly without affecting main conversation.
```
