# Claude Updates Data

**Purpose:** Track Claude Code changelog scans and research reports.

---

## Directory Structure

```
claude_updates/
├── last_scan.json          # Last changelog scan state (auto-generated)
└── research/               # Research reports from /claude-research
    └── [feature-name].md   # Individual feature research reports
```

---

## Files

### last_scan.json

Auto-generated by `/claude-updates` command. Tracks:
- Last scan timestamp
- Relevant changelog entries
- Pending documentation updates
- Action history

**Format:**
```json
{
  "last_scan": "ISO timestamp",
  "scan_range": {"from": "date", "to": "date"},
  "summary": {"total": N, "relevant": M, "breaking": X, "beta": Y},
  "relevant_entries": [...],
  "pending_actions": {...}
}
```

### research/

Markdown reports from `/claude-research` command.

**Naming convention:** `{feature-name}-{date}.md`

**Example:** `mcp-authentication-2025-11-30.md`

---

## Usage

**Scan for Claude Code updates:**
```bash
/claude-updates              # Check recent updates
/claude-updates 2025-11-01   # Scan since specific date
```

**Research a feature:**
```bash
/claude-research "MCP server authentication"
```

Reports saved to `research/` directory.

---

## Maintenance

- `last_scan.json` updates automatically with each scan
- Research reports accumulate over time (archive annually if needed)
- Part of sys-admin monthly maintenance cycle

---

## Related Commands

- `/claude-updates` - Scan Claude Code changelog
- `/claude-research` - Deep dive into features
- `/pacman` - System package updates
