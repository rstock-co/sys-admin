## Summary

v2.0.60 introduces **background agent support** and the **`--disable-slash-commands`** CLI flag - two features that fundamentally improve headless and automation workflows. Background agents enable true multitasking (fire off an agent, continue working, get results later), while `--disable-slash-commands` ensures clean, predictable behavior for scripting and CI/CD scenarios. This release also adds `/mcp enable` and `/mcp disable` commands for quick MCP server toggling.

---

## What Changed

- **Background Agent Support**: Agents now run asynchronously in the background while you continue working
- **`--disable-slash-commands` Flag**: New CLI flag to disable all slash commands for the current session
- **`--agent` Flag**: Override the agent setting for the current session
- **`/mcp enable [server-name]`**: Quickly enable specific MCP servers
- **`/mcp disable [server-name]`**: Quickly disable specific MCP servers
- **Enhanced Commit Attribution**: Model name now appears in "Co-Authored-By" commit messages (e.g., "Co-Authored-By: Claude Sonnet 4.5")
- **Fetch Optimization**: Skip summarization for pre-approved websites, reducing latency
- **VSCode Multi-Client**: Support for multiple terminal clients connecting to IDE server simultaneously
- **Agent Setting**: Configure main thread with specific agent's system prompt, tool restrictions, and model

## Deep Dive: `--disable-slash-commands` for Headless Operation

### The Problem It Solves

In headless/automation scenarios, slash commands introduce unpredictability:

1. **Accidental invocation**: If a prompt happens to start with `/`, it could trigger a slash command instead of being treated as a prompt
2. **Script interference**: Automation scripts passing prompts via `-p` flag risk unintended command execution
3. **CI/CD pipelines**: Build systems need deterministic behavior without Claude Code's interactive features

### How It Works

```bash
# Disable all slash commands for this session
claude --disable-slash-commands -p "Your prompt here"

# Combined with other automation flags
claude --disable-slash-commands \
       --dangerously-skip-permissions \
       --output-format json \
       -p "Analyze codebase for security issues"
```

### When to Use It

| Scenario | Use `--disable-slash-commands` |
|----------|--------------------------------|
| Interactive development | No |
| CI/CD pipelines | Yes |
| Automated scripts | Yes |
| Headless mode (`-p`) | Recommended |
| Background agent spawning | Yes |
| Autonomous agent operation | Yes |

### Key Characteristics

- **Session-scoped**: Only affects the current invocation
- **No configuration change**: Doesn't modify default settings
- **Combines with other flags**: Works alongside `--dangerously-skip-permissions`, `--output-format`, etc.
- **Prevents ALL slash commands**: Including custom ones defined in `.claude/commands/`

### Comparison: With vs Without

**Without `--disable-slash-commands`:**
```bash
# This could accidentally trigger /commit if user input starts with /commit
claude -p "/commit the changes"  # Might invoke slash command
```

**With `--disable-slash-commands`:**
```bash
# Treated as literal prompt text, not command
claude --disable-slash-commands -p "/commit the changes"  # Sent as prompt
```

## Deep Dive: Background Agents

### The Problem It Solves

Before v2.0.60, launching an agent for a complex task blocked the entire Claude Code session. You had to wait for completion before doing anything else. For lengthy operations (codebase exploration, refactoring analysis, multi-file reviews), this was a significant productivity bottleneck.

### How It Works

1. **Launch agent**: Start an agent task normally
2. **Background it**: Press `Ctrl+B` while the agent is running
3. **Continue working**: Do other tasks while agent runs
4. **Get results**: Agent hooks back in when complete

### Key Commands

| Action | Method |
|--------|--------|
| Background running task | `Ctrl+B` (or `Ctrl+B Ctrl+B` in tmux) |
| View background jobs | `/bashes` |
| View all background tasks | `/tasks` |
| Kill background process | Press `K` |
| Disable background tasks | Set `CLAUDE_CODE_DISABLE_BACKGROUND_TASKS=1` |

### Limitations

- **Permission inheritance**: Background subagents inherit parent's permissions
- **Auto-deny unknown permissions**: If a background subagent needs a permission it doesn't have, that tool call fails (but subagent continues)
- **No MCP tools**: MCP tools are not available in background subagents
- **No clarifying questions**: Background agents can't ask for user input

## Deep Dive: MCP Server Toggles

### The Problem It Solves

Before v2.0.60, toggling MCP servers required editing configuration files. This was problematic for:
- Testing unstable/new MCP servers
- Debugging server issues
- Context management (disabling irrelevant servers)
- Switching between workflows that need different server sets

### Usage

```bash
# Enable a specific server
/mcp enable playwright

# Disable a specific server
/mcp disable playwright

# Quick toggling for context management
/mcp disable browser-automation
/mcp enable database-tools
```

### Benefits for Automation

- **Programmatic control**: Can be scripted via hooks
- **Reduced configuration overhead**: No file editing needed
- **Session-specific**: Doesn't permanently change settings

## Symbiont Impact

### For Autonomous Agent Spawning

The `--disable-slash-commands` flag is **essential** for clean autonomous operation:

```bash
# Spawning a child agent autonomously
alacritty -e claude \
  --disable-slash-commands \
  --dangerously-skip-permissions \
  --working-directory /path/to/project \
  -p "Execute task from database"
```

Without this flag, malformed prompts (especially those injected from external systems like PostgreSQL) could accidentally trigger slash commands.

### For Headless Mode

Combined with other headless flags, enables fully autonomous pipelines:

```bash
# Complete headless automation pattern
claude \
  --disable-slash-commands \
  --dangerously-skip-permissions \
  --output-format json \
  --allowedTools Read,Grep,Glob,Write,Edit \
  --max-turns 10 \
  -p "$TASK_PROMPT" > result.json
```

### Background Agents for Parallel Workflows

For Symbiont's multi-specialist architecture:

```
Mycelia orchestrating parallel tasks:
├── Background: Health-coach analyzing sleep data
├── Background: Business-savant reviewing finances
├── Foreground: Direct user interaction
└── Results: All agents report back when complete
```

### MCP Toggle for Context Control

Dynamic MCP management based on task:

```bash
# Before health analysis task
/mcp disable business-tools
/mcp enable health-apis

# Before financial review
/mcp disable health-apis
/mcp enable business-tools
```

### Critical Patterns

**1. Clean Autonomous Spawn:**
```bash
claude --disable-slash-commands \
       --working-directory "$SPECIALIST_DIR"
```

**2. CI/CD Pipeline:**
```bash
claude --disable-slash-commands \
       --dangerously-skip-permissions \
       --output-format json \
       -p "Run security audit"
```

**3. Hook-Injected Tasks:**
SessionStart hook fetches instructions from database, passes to Claude without risk of slash command interference because `--disable-slash-commands` is set.

## Configuration Examples

### Fully Autonomous Agent

```bash
#!/bin/bash
# spawn-autonomous.sh

export CLAUDE_CODE_DISABLE_BACKGROUND_TASKS=0  # Allow backgrounds
unset ANTHROPIC_API_KEY  # Use Max subscription

claude \
  --disable-slash-commands \
  --dangerously-skip-permissions \
  --working-directory "$1" \
  -p "$2"
```

### CI Pipeline Integration

```yaml
# .github/workflows/claude-review.yml
- name: Claude Security Review
  run: |
    claude \
      --disable-slash-commands \
      --dangerously-skip-permissions \
      --output-format json \
      -p "Review code changes for security vulnerabilities" \
      > security-report.json
```
