## Deep Dive: --max-budget-usd for Cost Control

### Purpose

The `--max-budget-usd` flag (or `maxBudgetUsd` in SDK) enables **hard spending limits** on autonomous agent execution. This is critical for safe autonomous operation - preventing runaway costs when agents operate without human oversight.

### SDK Configuration

#### TypeScript
```typescript
import { query } from "@anthropic-ai/claude-agent-sdk";

const result = await query({
  prompt: "Refactor this codebase for better performance",
  options: {
    maxBudgetUsd: 5.00,  // Hard limit: $5.00
    permissionMode: "bypassPermissions",
    allowedTools: ["Read", "Edit", "Bash", "Glob", "Grep"]
  }
});
```

#### Python
```python
import asyncio
from claude_agent_sdk import query, ClaudeAgentOptions

async def main():
    async for message in query(
        prompt="Refactor this codebase for better performance",
        options=ClaudeAgentOptions(
            max_budget_usd=5.00,  # Hard limit: $5.00
            permission_mode="bypassPermissions",
            allowed_tools=["Read", "Edit", "Bash", "Glob", "Grep"]
        )
    ):
        print(message)

asyncio.run(main())
```

### Behavior When Budget Exceeded

When the budget is exceeded, the SDK:

1. **Stops execution immediately** - No further API calls are made
2. **Returns error result** - Result message has `subtype: 'error_max_budget_usd'`
3. **Reports usage** - `total_cost_usd` and `modelUsage` fields are populated

```typescript
// Result message structure when budget exceeded
{
  type: 'result',
  subtype: 'error_max_budget_usd',
  session_id: '...',
  duration_ms: 45000,
  total_cost_usd: 5.02,
  usage: { input_tokens: 150000, output_tokens: 12000, ... },
  modelUsage: {
    'claude-sonnet-4-5': { costUSD: 4.50, ... },
    'claude-haiku-4-5': { costUSD: 0.52, ... }
  },
  errors: ['Budget exceeded: $5.02 > $5.00 limit']
}
```

### Cost Tracking Fields

The SDK provides comprehensive cost tracking:

| Field | Location | Description |
|-------|----------|-------------|
| `total_cost_usd` | Result message | Total cost of session (authoritative) |
| `modelUsage` | Result message | Per-model breakdown with costUSD |
| `usage` | Each assistant message | Per-turn token usage |

## Deep Dive: Agent Resume for Context Preservation

### The Problem

Prior to v2.0.28, every subagent invocation started fresh - no memory of previous work. For complex multi-step workflows, this meant:

- Repeated context gathering (costly)
- Lost reasoning chains
- Inconsistent approaches across invocations

### The Solution: Resume Capability

Claude can now **choose to resume** a previously created subagent, maintaining full conversation history including:

- All previous tool calls and results
- Reasoning and analysis from prior work
- Files read and changes made

### How It Works

1. **Subagent completes** - Claude receives the agent ID
2. **Subsequent request** - Claude decides whether to start fresh or resume
3. **If resumed** - Full context is restored from the transcript file

### Example Workflow

```
User: "Use the code-reviewer subagent to review the authentication module"
[Subagent completes, ID: agent-abc123]

User: "Continue that code review and now analyze the authorization logic"
[Claude resumes agent-abc123 with full context from previous conversation]
```

### Transcript Persistence

Subagent transcripts are stored at:
```
~/.claude/projects/{project}/{sessionId}/subagents/agent-{agentId}.jsonl
```

Key characteristics:
- **Independent of main conversation** - Compaction doesn't affect subagent transcripts
- **Session-scoped** - Persist within the session
- **Auto-cleanup** - Removed after `cleanupPeriodDays` (default: 30)

### SDK Access to Agent IDs

In SDK, track agent IDs from SubagentStart/SubagentStop hooks:

```typescript
hooks: {
  SubagentStop: [{
    hooks: [async (input) => {
      console.log(`Agent ${input.agent_id} completed`);
      console.log(`Transcript: ${input.agent_transcript_path}`);
      return {};
    }]
  }]
}
```

## Deep Dive: Model Selection for Subagents

### Dynamic Model Selection

Claude can now **automatically choose** which model (Haiku, Sonnet, or Opus) each subagent uses based on task complexity. This optimizes cost vs. capability without manual configuration.

### Available Model Options

| Model | Use Case | Cost Level |
|-------|----------|------------|
| `haiku` | Simple tasks, fast lookups, routine operations | Low (~3x cheaper) |
| `sonnet` | Standard coding, most development work | Medium |
| `opus` | Complex architecture, security audits, critical decisions | High |
| `inherit` | Same as main conversation (default) | Varies |

### Configuration in Subagent Frontmatter

```yaml
---
name: quick-lookup
description: Fast file discovery and simple searches
tools: Read, Glob, Grep
model: haiku
---

You are a fast lookup agent. Find files and extract simple information quickly.
```

### Recommended Model Routing Patterns

**By Task Complexity:**
- **Haiku** - Basic data lookups, documentation generation, standard responses
- **Sonnet** - Code review, refactoring, test writing, most development tasks
- **Opus** - Security auditing, architecture decisions, complex reasoning

**Orchestrator + Workers Pattern:**
```
Sonnet 4.5 (orchestrator) → Complex task decomposition, coordination
    ├── Haiku 4.5 (worker) → Subtask 1 (90% of Sonnet capability, 3x cheaper)
    ├── Haiku 4.5 (worker) → Subtask 2
    └── Haiku 4.5 (worker) → Subtask 3
```

This pattern achieves:
- 2-2.5x overall cost reduction
- 85-95% quality maintenance
- Faster execution (parallel Haiku workers)

### Cost Savings Example

From the Digital Marketing Guide research:

> "Plan Mode, sub-agent resumption, and dynamic model selection—enabling marketing teams to automate content audits, keyword research, and campaign analysis with 75% time savings and 70% cost reduction through intelligent Haiku/Sonnet/Opus allocation."

## Symbiont Impact: Autonomous Operation Cost Control

### Why This Matters

For symbiont-style autonomous operation, these features address critical safety requirements:

| Requirement | Solution | Feature |
|-------------|----------|---------|
| Prevent runaway costs | Hard budget limits | `maxBudgetUsd` |
| Preserve context across sessions | Resume capability | Agent resume |
| Optimize cost efficiency | Smart model routing | Dynamic model selection |
| Maintain observability | Lifecycle tracking | SubagentStop hooks (v2.0.42) |

### Implementation Pattern: Safe Autonomous Worker

```typescript
import { query } from "@anthropic-ai/claude-agent-sdk";

async function runAutonomousTask(task: string, budgetUsd: number) {
  let totalCost = 0;

  const result = await query({
    prompt: task,
    options: {
      // Cost control
      maxBudgetUsd: budgetUsd,

      // Full autonomy
      permissionMode: "bypassPermissions",
      allowDangerouslySkipPermissions: true,

      // Smart routing via subagents
      agents: {
        "quick-lookup": {
          description: "Fast file searches",
          prompt: "Find files quickly.",
          model: "haiku",
          tools: ["Read", "Glob", "Grep"]
        },
        "code-worker": {
          description: "Code modifications",
          prompt: "Implement code changes.",
          model: "sonnet",
          tools: ["Read", "Edit", "Write", "Bash"]
        }
      },

      // Lifecycle hooks for logging
      hooks: {
        SubagentStop: [{
          hooks: [async (input) => {
            console.log(`[LEDGER] Agent ${input.agent_type} completed`);
            return {};
          }]
        }]
      }
    }
  });

  // Check result
  if (result.subtype === 'error_max_budget_usd') {
    console.warn(`Budget exceeded: $${result.total_cost_usd}`);
    // Handle gracefully - work was stopped safely
  }

  return result;
}
```

### Combining All Features

A complete autonomous operation setup:

```typescript
// Phase 1: Planning (low budget, Opus for quality)
const plan = await query({

  }
```

## Known Limitations

1. **Budget is per-session** - Cannot carry over unused budget across sessions
2. **Resume requires same session** - Cannot resume agents from previous sessions
3. **Model selection is per-agent** - Cannot change model mid-execution
4. **No budget warnings** - Only notified when budget is exceeded (no threshold alerts)
