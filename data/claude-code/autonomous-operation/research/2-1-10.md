## Summary

v2.1.10 introduces the **Setup hook** - a new lifecycle event that fires during repository initialization and maintenance operations. Unlike SessionStart (which runs every session), Setup hooks require explicit CLI flags (`--init`, `--init-only`, or `--maintenance`) and are designed for one-time or periodic operations like dependency installation, migrations, or environment configuration.

---

## What Changed

- **Setup Hook Event**: New hook that fires with `--init`, `--init-only`, or `--maintenance` flags
- **OAuth Copy Shortcut**: Keyboard shortcut 'c' to copy OAuth URL when browser fails to open
- **Heredoc Fix**: Fixed crash with heredocs containing JavaScript template literals like `${index + 1}`
- **Keystroke Capture**: Improved startup to capture keystrokes typed before REPL initializes
- **File Suggestions**: Display as removable attachments instead of inserting raw text
- **VSCode Plugin Display**: Install counts shown on plugin listings
- **Trust Warnings**: Appear when installing plugins for security awareness

## Deep Dive: Setup Hook

### When It Fires

| Flag | Behavior |
|------|----------|
| `--init` | Run Setup hooks, then start interactive mode |
| `--init-only` | Run Setup hooks and exit (no interactive session) |
| `--maintenance` | Run Setup hooks with maintenance trigger and exit |

### Input JSON Structure

```json
{
  "session_id": "abc123",
  "transcript_path": "~/.claude/projects/.../session.jsonl",
  "cwd": "/path/to/project",
  "permission_mode": "default",
  "hook_event_name": "Setup",
  "trigger": "init"
}
```

The `trigger` field is either `"init"` (from `--init` or `--init-only`) or `"maintenance"` (from `--maintenance`).

### Matchers

```json
{
  "hooks": {
    "Setup": [
      {
        "matcher": "init",
        "hooks": [{ "type": "command", "command": "./scripts/init.sh" }]
      },
      {
        "matcher": "maintenance",
        "hooks": [{ "type": "command", "command": "./scripts/maintenance.sh" }]
      }
    ]
  }
}
```

### CLAUDE_ENV_FILE Access

Like SessionStart, Setup hooks can persist environment variables:

```bash
#!/bin/bash
# Setup hook script

if [ -n "$CLAUDE_ENV_FILE" ]; then
  echo 'export NODE_ENV=development' >> "$CLAUDE_ENV_FILE"
  echo 'export DATABASE_URL=postgresql://...' >> "$CLAUDE_ENV_FILE"
fi

# Install dependencies
npm install

exit 0
```

Variables written to `CLAUDE_ENV_FILE` are available in all subsequent Bash commands during the session.

### Output Format

Setup hooks can inject context via JSON output:

```json
{
  "hookSpecificOutput": {
    "hookEventName": "Setup",
    "additionalContext": "Repository initialized with Node 20 and PostgreSQL connection"
  }
}
```

Multiple hooks' `additionalContext` values are concatenated.

### Setup vs SessionStart

| Aspect | Setup Hook | SessionStart Hook |
|--------|------------|-------------------|
| **When** | Only with `--init`/`--maintenance` flags | Every session start |
| **Purpose** | One-time or periodic operations | Every-session context loading |
| **Performance** | Expensive operations OK | Keep fast |
| **Use Cases** | Install deps, run migrations, cleanup | Load context, set env vars |

## How to Use It

### Basic Configuration

```json
{
  "hooks": {
    "Setup": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "$CLAUDE_PROJECT_DIR/.claude/hooks/setup.sh",
            "timeout": 120
          }
        ]
      }
    ]
  }
}
```

### Initialization Script Example

```bash
#!/bin/bash
# .claude/hooks/setup.sh

set -e

echo "Initializing project environment..."

# Install dependencies
if [ -f "package.json" ]; then
  npm install
fi

if [ -f "requirements.txt" ]; then
  pip install -r requirements.txt
fi

# Run migrations
if [ -f "prisma/schema.prisma" ]; then
  npx prisma migrate dev
fi

# Set up environment variables
if [ -n "$CLAUDE_ENV_FILE" ]; then
  echo 'export PATH="$PATH:./node_modules/.bin"' >> "$CLAUDE_ENV_FILE"
fi

echo "Setup complete!"
exit 0
```

### Maintenance Script Example

```bash
#!/bin/bash
# .claude/hooks/maintenance.sh

echo "Running maintenance tasks..."

# Clean build artifacts
rm -rf dist/ .next/ __pycache__/

# Update dependencies
npm update

# Prune docker
docker system prune -f

# Vacuum database
psql -c "VACUUM ANALYZE"

echo "Maintenance complete!"
exit 0
```

### Spawning with Setup

```bash
# Initialize new project
claude --init

# Run maintenance and exit
claude --maintenance

# Initialize only, no interactive session
claude --init-only
```

## Symbiont Impact

### For Autonomous Agent Spawning

The Setup hook provides a **one-time initialization point** distinct from SessionStart. For autonomous operation:

**Spawning Pattern with Setup:**
```bash
# First-time specialist initialization
claude --init-only --working-directory ~/symbiont/mycelium/new-specialist/

# Subsequent launches use SessionStart for context injection
claude --working-directory ~/symbiont/mycelium/new-specialist/
```

### SGTA Factory Integration

When creating new specialists via the SGTA factory:

1. **Setup hook** runs on first `--init`:
   - Create symlinks to `core/`
   - Initialize CLAUDE.md from template
   - Register in Queen's Ledger
   - Configure MCP connections

2. **SessionStart hook** runs on subsequent sessions:
   - Inject instructions from database
   - Load specialist context

### Configuration Example for Specialists

```json
{
  "hooks": {
    "Setup": [
      {
        "matcher": "init",
        "hooks": [{
          "type": "command",
          "command": "python3 ~/.claude/hooks/init-specialist.py"
        }]
      }
    ],
    "SessionStart": [
      {
        "matcher": "startup",
        "hooks": [{
          "type": "command",
          "command": "python3 ~/.claude/hooks/inject-instructions.py"
        }]
      }
    ]
  }
}
```

### Maintenance for Overnight Daemon

Use `--maintenance` for periodic cleanup:

```bash
# Cron job for nightly maintenance
0 3 * * * cd ~/symbiont && claude --maintenance
```

Maintenance hook could:
- Clean up old session files
- Vacuum PostgreSQL
- Update specialist registrations
- Prune stale tasks

### Limitations

1. **Requires explicit flag**: Won't run automatically - must use `--init`, `--init-only`, or `--maintenance`
2. **No matchers for SessionStart-style sources**: Only `init` and `maintenance` matchers, not `startup`/`resume`/`clear`/`compact`
3. **Exit code 2 shows to user only**: Unlike PreToolUse, Setup hook errors don't feed back to Claude
